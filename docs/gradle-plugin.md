# Gradle Plugin for the HTMA framework
This plugin is a crucial part of the HTMA framework.
You will most likely need it in order to use the framework effectively.

Install this plugin by including it in your plugins section in your `build.gradle.kts` file:

```kotlin
plugins {
  id("de.jakobschaefer.ktor-server-htma") version "<currentVersion>"
}
```

## How does it work
The HTMA Ktor plugin requires a WEB-INF folder within your
class loader resources. By convention this folder must have the following structure:
```code
/WEB-INF
  /.app        // App manifest generated by this gradle plugin
  /.vite       // Metafiles generated by the vite build tool
  /web         // HTML templates
  /assets      // Assets, also generated by vite e.g. js-libs or fonts
```

This plugin helps you to build the proper WEB-INF resource folder by leveraging
the famous [gradle-node-plugin](https://github.com/node-gradle/gradle-node-plugin)
and [vite](https://vitejs.dev/).
You will need a proper `package.json` and `vite.config.js` next to your gradle files.

Use the [example-site](/example-site) to bootstrap your own project.

## Tasks
This plugin also applies the [gradle-node-plugin](https://github.com/node-gradle/gradle-node-plugin).
All tasks provided by this plugin will also be available.

Additionally, there are the following tasks added by this plugin:

* **npxViteBuild**:\
  [NpxTask](https://github.com/node-gradle/gradle-node-plugin/blob/main/docs/usage.md#executing-npm-commands-via-npx) - executes `npx vite build`.
* **buildAppManifest:**\
  Extracts the [app manifest](/gradle-plugin/src/main/kotlin/de/jakobschaefer/htma/gradle/AppManifest.kt) from the `web` folder.

These tasks will be run automatically on build.
